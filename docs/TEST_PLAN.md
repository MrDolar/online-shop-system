# 测试计划

## 1. 测试概述

### 1.1 测试目标
验证在线商城系统所有功能模块的正确性、稳定性和安全性。

### 1.2 测试范围
- 用户认证模块
- 商品管理模块
- 购物车模块
- 订单管理模块
- 支付模块
- AI 功能模块
- 数据统计模块

### 1.3 测试环境
- Python 3.11+
- pytest 7.x
- SQLite（测试数据库）
- httpx（异步测试客户端）

## 2. 测试用例

### 2.1 用户认证模块

#### TC-AUTH-001: 用户注册 - 成功
- **前置条件**：无
- **输入**：有效的用户名、邮箱、密码
- **预期结果**：返回 201，用户信息正确

#### TC-AUTH-002: 用户注册 - 用户名重复
- **前置条件**：已存在同名用户
- **输入**：已存在的用户名
- **预期结果**：返回 400，提示用户名已存在

#### TC-AUTH-003: 用户注册 - 邮箱重复
- **前置条件**：已存在同邮箱用户
- **输入**：已存在的邮箱
- **预期结果**：返回 400，提示邮箱已存在

#### TC-AUTH-004: 用户登录 - 成功
- **前置条件**：已注册用户
- **输入**：正确的用户名和密码
- **预期结果**：返回 200，包含 access_token

#### TC-AUTH-005: 用户登录 - 密码错误
- **前置条件**：已注册用户
- **输入**：错误的密码
- **预期结果**：返回 401

#### TC-AUTH-006: 用户登录 - 用户不存在
- **前置条件**：无
- **输入**：不存在的用户名
- **预期结果**：返回 401

#### TC-AUTH-007: 获取用户信息
- **前置条件**：已登录
- **输入**：有效的 Token
- **预期结果**：返回 200，用户信息正确

#### TC-AUTH-008: 获取用户信息 - 未认证
- **前置条件**：无
- **输入**：无 Token
- **预期结果**：返回 401

#### TC-AUTH-009: 更新用户信息
- **前置条件**：已登录
- **输入**：新的姓名、电话、地址
- **预期结果**：返回 200，信息已更新

### 2.2 商品管理模块

#### TC-PROD-001: 获取商品列表
- **前置条件**：数据库中有商品
- **输入**：无参数
- **预期结果**：返回 200，商品列表

#### TC-PROD-002: 按分类筛选商品
- **前置条件**：存在分类和商品
- **输入**：category_id
- **预期结果**：仅返回该分类的商品

#### TC-PROD-003: 按关键词搜索
- **前置条件**：存在商品
- **输入**：keyword
- **预期结果**：返回名称或描述包含关键词的商品

#### TC-PROD-004: 按价格区间筛选
- **前置条件**：存在商品
- **输入**：min_price, max_price
- **预期结果**：仅返回价格在区间内的商品

#### TC-PROD-005: 商品排序
- **前置条件**：存在多个商品
- **输入**：sort_by=price_asc
- **预期结果**：按价格升序排列

#### TC-PROD-006: 分页
- **前置条件**：存在多个商品
- **输入**：page=2, page_size=5
- **预期结果**：返回第 6-10 个商品

#### TC-PROD-007: 获取商品详情
- **前置条件**：商品存在
- **输入**：product_id
- **预期结果**：返回 200，商品详情

#### TC-PROD-008: 获取商品详情 - 不存在
- **前置条件**：无
- **输入**：不存在的 product_id
- **预期结果**：返回 404

#### TC-PROD-009: 创建商品（管理员）
- **前置条件**：管理员已登录
- **输入**：商品信息
- **预期结果**：返回 201，商品创建成功

#### TC-PROD-010: 创建商品（普通用户）
- **前置条件**：普通用户已登录
- **输入**：商品信息
- **预期结果**：返回 403

#### TC-PROD-011: 更新商品
- **前置条件**：管理员已登录，商品存在
- **输入**：更新数据
- **预期结果**：返回 200，商品已更新

#### TC-PROD-012: 删除商品
- **前置条件**：管理员已登录，商品存在
- **输入**：product_id
- **预期结果**：返回 204

### 2.3 购物车模块

#### TC-CART-001: 获取购物车 - 空购物车

- **输入**：无

#### TC-CART-003: 添加已有商品到购物车

- **输入**：同一 product_id

#### TC-CART-004: 添加商品 - 库存不足

- **预期结果**：返回 400

#### TC-CART-005: 更新购物车商品数量
- **前置条件**：购物车有商品
- **输入**：新数量
- **预期结果**：数量已更新

#### TC-CART-006: 删除购物车商品
- **前置条件**：购物车有商品

- **预期结果**：返回 204

#### TC-CART-007: 清空购物车
- **前置条件**：购物车有商品
- **输入**：无
- **预期结果**：返回 204，购物车为空

## 3. 测试执行

### 3.1 运行测试

```bash
# 安装测试依赖
pip install pytest pytest-asyncio httpx

# 运行所有测试
pytest tests/ -v

# 运行特定模块
pytest tests/test_auth.py -v
pytest tests/test_products.py -v

# 生成覆盖率报告
pytest tests/ --cov=app --cov-report=html
```

### 3.2 测试数据准备

测试使用独立的 SQLite 数据库，每个测试用例前后清理数据。
