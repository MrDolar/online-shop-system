# 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
在线商城系统 (Online Shop System)

### 1.2 产品定位
一个功能完整的在线购物平台，集成 AI 智能功能，为用户提供便捷的购物体验和智能化服务。

### 1.3 目标用户
- **普通用户**：浏览商品、下单购买、管理订单
- **管理员**：管理商品、查看统计数据、维护系统

## 2. 功能需求

### 2.1 用户系统 (P0)

#### 2.1.1 用户注册
- 用户名、邮箱、密码注册
- 用户名和邮箱唯一性校验
- 密码加密存储

#### 2.1.2 用户登录
- 用户名/密码登录
- JWT Token 认证
- Token 过期机制（24小时）

#### 2.1.3 个人资料管理
- 查看个人信息
- 修改姓名、电话、地址

### 2.2 商品管理 (P0)

#### 2.2.1 商品浏览
- 商品列表展示（分页）
- 商品详情查看
- 按分类筛选
- 按关键词搜索
- 按价格区间筛选
- 多种排序方式（价格、销量、评分、时间）

#### 2.2.2 商品管理（管理员）
- 创建商品
- 编辑商品信息
- 上下架管理
- 库存管理
- 删除商品

#### 2.2.3 商品分类
- 分类列表
- 分类创建（管理员）
- 分类删除（管理员）

### 2.3 购物车 (P0)

#### 2.3.1 购物车操作
- 添加商品到购物车
- 修改商品数量
- 删除购物车商品
- 清空购物车

#### 2.3.2 购物车展示
- 商品名称、价格、数量
- 小计金额
- 总金额计算

### 2.4 订单管理 (P0)

#### 2.4.1 订单创建
- 从购物车生成订单
- 填写收货地址、联系电话
- 订单备注
- 库存校验与扣减

#### 2.4.2 订单状态
- 待支付 (pending)
- 已支付 (paid)
- 已发货 (shipped)
- 已送达 (delivered)
- 已取消 (cancelled)

#### 2.4.3 订单操作
- 查看订单列表
- 查看订单详情
- 取消订单（仅待支付状态）

### 2.5 支付模拟 (P0)

#### 2.5.1 支付方式
- 支付宝（模拟）
- 微信支付（模拟）
- 信用卡（模拟）

#### 2.5.2 支付流程
- 发起支付请求
- 模拟支付成功
- 更新订单状态
- 生成交易流水号

### 2.6 商品评论 (P1)

#### 2.6.1 评论功能
- 发表评论（文字 + 评分 1-5）
- 查看商品评论列表
- 自动计算商品平均评分

### 2.7 数据统计 (P1)

#### 2.7.1 销售统计
- 总订单数、总收入
- 今日订单数、今日收入
- 平均订单金额

#### 2.7.2 商品统计
- 商品总数、上架数
- 低库存商品数
- 销量排行榜

#### 2.7.3 用户统计
- 用户总数
- 今日新增用户
- 活跃用户数

#### 2.7.4 销售趋势
- 近 N 天销售趋势图数据
- 分类销售统计

### 2.8 AI 功能 (P1)

#### 2.8.1 智能商品推荐
- 基于用户购买历史推荐
- 新用户推荐热门商品
- 返回推荐理由

#### 2.8.2 情感分析
- 分析评论情感倾向（正面/负面/中性）
- 返回情感分数
- 提供分析说明

#### 2.8.3 智能客服
- 对话式交互
- 记录对话上下文
- 回答商品、订单、支付相关问题
- 超出能力范围时建议联系人工客服

## 3. 非功能需求

### 3.1 性能要求
- API 响应时间 < 500ms（AI 功能除外）
- 支持并发用户 > 100

### 3.2 安全要求
- 密码加密存储（bcrypt）
- JWT Token 认证
- 管理员权限控制
- 环境变量存储敏感信息

### 3.3 可用性
- RESTful API 设计
- 完整的 API 文档（Swagger/ReDoc）
- 清晰的错误提示

## 4. 技术约束

- 后端：Python FastAPI
- 数据库：SQLite（开发）/ 可扩展至 PostgreSQL
- ORM：SQLAlchemy
- 认证：JWT
- AI：OpenAI Compatible API
- 前端：React（占位）
